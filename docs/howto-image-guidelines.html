コーポレートサイト - 画像とディレクトリ構造ガイドライン

バージョン: 2.0 | 2026年5月11日

作成: Web開発チーム

1. はじめに

このドキュメントは、コーポレートサイトの全記事コンテンツ（howto、manual、insights）の画像管理とディレクトリ構造に関するガイドラインです。 一貫した構造と命名規則を採用することで、コンテンツの管理と保守が容易になります。

1.1 対象コンテンツ

大カテゴリ小カテゴリ内容

manualwixWix Studio 運用マニュアル
manualshopifyShopifyマニュアル（今後追加）
insightsec-growthEC成長・Shopify構築ナレッジ
insightsweb-marketingWebマーケティングナレッジ

2. コンテンツディレクトリ構造

記事HTMLファイルは以下の構造で管理されています：

/howto/ ├── posts/ │ ├── article/ # ノウハウ記事 │ │ ├── article1/ │ │ │ └── index.html │ │ └── article2/ │ │ └── index.html │ └── manual/ # マニュアル │ ├── wix1/ │ │ └── index.html │ └── wix2/ │ └── index.html ├── templates/ # テンプレート │ ├── unified-article-template.html │ └── unified-manual-template.html └── allpost/ # 全記事一覧 /insights/ ├── ec-growth/ # EC成長テーマ │ ├── index.html # ピラー記事 │ ├── a-1/ │ │ └── index.html │ └── b-1/ │ └── index.html └── web-marketing/ # Webマーケテーマ ├── index.html # ピラー記事 └── a-1/ └── index.html

3. 画像ディレクトリ構造

画像は以下の2階層（大カテゴリ/小カテゴリ）で管理します。小カテゴリ直下にフラット配置します。

/assets/images/ ├── howto/ │ └── article/ # ノウハウ記事画像 │ ├── article1_thumb.png │ ├── article1_section1.png │ ├── article2_thumb.png │ └── ... │ ├── manual/ │ ├── wix/ # Wixマニュアル画像 │ │ ├── wix1_thumb.webp │ │ ├── wix1_section1.png │ │ └── ... │ └── shopify/ # Shopifyマニュアル画像（今後） │ └── insights/ ├── ec-growth/ # EC成長テーマ画像 │ ├── pillar_thumb.webp │ ├── pillar_section1.webp │ ├── a-1_thumb.webp │ └── ... └── web-marketing/ # Webマーケテーマ画像 ├── pillar_thumb.webp ├── a-1_thumb.webp ├── a-1_section1.webp └── ...

重要: 記事ごとのサブフォルダは作成しません。小カテゴリ直下にすべての画像をフラットに配置してください。

4. 画像命名規則

4.1 基本パターン

/assets/images/[大カテゴリ]/[小カテゴリ]/[記事ID]_[用途].[拡張子]

セパレータはアンダースコア _ を使用します。ハイフン - は記事ID内のみで使用可能です（例: a-1）。

4.2 用途別の命名パターン

用途 命名パターン 説明
サムネイル [記事ID]_thumb.[拡張子] 記事一覧やカード表示用のサムネイル画像
セクション画像 [記事ID]_section[番号].[拡張子] 記事内の各セクションに使用するキャプション画像
共有プレースホルダ shared_main.[拡張子] 個別画像が未設定の記事で使用する共通画像
ピラー記事専用 pillar_[用途].[拡張子] insightsピラー記事の画像（pillar_thumb, pillar_section1 等）

/assets/images/howto/article/article3_thumb.webp

/assets/images/insights/ec-growth/pillar_thumb.webp

禁止事項:

  • 日本語のファイル名・フォルダ名（例: Shopify導入事例.webp → NG）
  • 記事ごとのサブフォルダ作成（例: ec-growth/クラスター記事A/ → NG）
  • セパレータにハイフンを使用（例: wix1-thumb.webp → NG、正しくは wix1_thumb.webp）
  • 大カテゴリを跨いだ画像参照（例: insights記事から /assets/images/howto/ を参照 → NG）

5. HTMLでの画像参照方法

5.1 howto/article

<img src="/assets/images/howto/article/[記事ID]_[用途].[拡張子]" alt="代替テキスト">

例：

<img src="/assets/images/howto/article/article1_thumb.png" alt="記事サムネイル">
<img src="/assets/images/howto/article/article1_section1.png" alt="セクション1の画像">

5.2 manual/wix

<img src="/assets/images/manual/wix/[記事ID]_[用途].[拡張子]" alt="代替テキスト">

例：

<img src="/assets/images/manual/wix/wix1_thumb.webp" alt="マニュアルサムネイル">
<img src="/assets/images/manual/wix/wix1_section1.png" alt="手順の説明画像">

5.3 insights/ec-growth

<img src="/assets/images/insights/ec-growth/[記事ID]_[用途].[拡張子]" alt="代替テキスト">

例：

<img src="/assets/images/insights/ec-growth/a-1_thumb.webp" alt="記事サムネイル">
<img src="/assets/images/insights/ec-growth/pillar_section1.webp" alt="ピラー記事画像">

5.4 insights/web-marketing

<img src="/assets/images/insights/web-marketing/[記事ID]_[用途].[拡張子]" alt="代替テキスト">

例：

<img src="/assets/images/insights/web-marketing/a-1_thumb.webp" alt="記事サムネイル">

6. JSデータファイルでの画像パス

記事一覧や関連記事の表示は、以下のJSデータファイルの image プロパティを参照します。

ファイル対象
/assets/js/howto.jshowto/article + manual の記事データ
/assets/js/insights.jsinsights の記事データ

新しい記事を追加する際は、対応するJSファイルにエントリを追加し、image プロパティに正しいパスを設定してください。

  {
    id: 'article7',
    title: '記事タイトル',
    image: '/assets/images/howto/article/article7_thumb.webp',
    ...
  }

7. パンくずリスト生成

パンくずリストは、記事タイプに基づいて動的に生成されます。

7.1 HTMLマークアップ

<div id="breadcrumb-container" data-page="[記事ID]"></div>

例：

<div id="breadcrumb-container" data-page="article1"></div>
<div id="breadcrumb-container" data-page="wix1"></div>
<div id="breadcrumb-container" data-page="ec-growth-a-1"></div>

8. テンプレートの使用方法

新しい記事を作成する際は、以下のテンプレートを使用してください：

種類テンプレート
ノウハウ記事/howto/templates/unified-article-template.html
マニュアル/howto/templates/unified-manual-template.html

8.1 新規記事作成の手順

  1. テンプレートファイルをコピー
  2. 記事ディレクトリを作成（例: /howto/posts/article/article7/）
  3. コピーしたファイルを index.html として保存
  4. 以下を更新：
    • title、meta description
    • data-page 属性（パンくず用）
    • 画像パス（命名規則に従う）
    • 記事コンテンツ
  5. 画像ファイルを /assets/images/[大カテゴリ]/[小カテゴリ]/ に配置
  6. JSデータファイル（howto.js or insights.js）にエントリを追加

9. ベストプラクティス

  • 推奨フォーマット：
    • サムネイル・セクション画像: WebP形式（軽量・高品質）
    • スクリーンショット: PNG形式（文字の可読性が重要な場合）
    • 写真: JPGまたはWebP形式
  • 推奨サイズ：
    • サムネイル: 幅400px程度
    • セクション画像: 幅800px程度
    • メイン画像: 幅1200px程度
  • すべての画像に適切な代替テキスト（alt属性）を設定
  • ファイル名には英数字・ハイフン・アンダースコアのみ使用（日本語・スペース禁止）
  • 他の大カテゴリのディレクトリから画像を参照しない

10. まとめ
