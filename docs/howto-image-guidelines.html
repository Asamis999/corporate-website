Howto BOX - 画像とディレクトリ構造ガイドライン

バージョン: 1.0 | 2025年8月25日

作成: Web開発チーム

1. はじめに

このドキュメントは、Howto BOXの記事（ノウハウ記事とマニュアル）の画像管理とディレクトリ構造に関するガイドラインです。 一貫した構造と命名規則を採用することで、コンテンツの管理と保守が容易になります。

2. 基本ディレクトリ構造

Howto BOXコンテンツは以下の基本構造に従って整理されています：

/howto/ ├── article/ # ノウハウ記事ディレクトリ │ ├── article1/ # 記事ID = article1 │ │ └── index.html │ ├── article2/ │ │ └── index.html │ └── article-template.html # ノウハウ記事テンプレート │ ├── manual/ # マニュアル記事ディレクトリ │ ├── manual1/ # マニュアルID = manual1 │ │ └── index.html │ └── manual-template.html # マニュアルテンプレート │ └── allpost/ # 全記事一覧 ├── article/ # ノウハウ記事一覧 ├── manual/ # マニュアル一覧 └── tag/ # タグ別一覧

各記事フォルダにはindex.htmlファイルが含まれ、これが記事のメインコンテンツです。

3. 画像ディレクトリ構造

画像は以下のディレクトリ構造で管理します：

/assets/ └── images/ ├── article/ # ノウハウ記事用の画像 │ ├── article1/ # article1の画像フォルダ │ │ ├── article1-main.png # メイン画像 │ │ ├── article1-section1.png # セクション1の画像 │ │ └── article1-section2.png # セクション2の画像 │ └── article2/ │ └── ... │ └── manual/ # マニュアル用の画像 ├── manual1/ # manual1の画像フォルダ │ ├── manual1-main.jpg # メイン画像 │ └── manual1-section1.jpg # セクション画像 └── manual2/ └── ...

注意: 画像ファイルは各記事IDに対応するフォルダ内に格納します。これにより記事と画像の関連性が明確になり、管理が容易になります。

4. 画像命名規則

画像ファイルは以下の命名規則に従ってください：

4.1 基本形式

[記事ID]-[用途].拡張子

例：

  • article1-main.png - article1のメイン画像
  • article1-section1.png - article1のセクション1の画像
  • manual2-thumbnail.jpg - manual2のサムネイル画像

4.2 特別な画像タイプ

タイプ 命名パターン 説明
メイン画像 [記事ID]-main.拡張子 記事のヘッダー画像、一覧表示のサムネイルとしても使用
セクション画像 [記事ID]-section[番号].拡張子 記事内の各セクションに使用する画像
サムネイル [記事ID]-thumbnail.拡張子 任意。特別なサムネイルが必要な場合（main画像と異なる場合）

重要: 画像ファイル名には必ず記事IDをプレフィックスとして含めてください。これにより、どの記事に属する画像かが一目で分かります。

5. HTMLでの画像参照方法

HTMLファイル内で画像を参照する際は、以下のパターンに従ってください：

5.1 ノウハウ記事内での参照

<img src="/assets/images/article/[記事ID]/[記事ID]-[用途].拡張子" alt="代替テキスト">

例：

<img src="/assets/images/article/article1/article1-main.png" alt="記事メイン画像">

5.2 マニュアル内での参照

<img src="/assets/images/manual/[マニュアルID]/[マニュアルID]-[用途].拡張子" alt="代替テキスト">

例：

<img src="/assets/images/manual/manual1/manual1-section2.jpg" alt="手順の説明画像">

6. パンくずリスト生成

パンくずリストは、記事タイプ（ノウハウ記事/マニュアル）に基づいて動的に生成されます。

6.1 HTMLマークアップ

各記事ページには以下のマークアップを含めます：

<div id="breadcrumb-container" data-page="[記事ID]"></div>

例：

<div id="breadcrumb-container" data-page="article1"></div>
<div id="breadcrumb-container" data-page="manual1" data-title="カスタムタイトル"></div>

6.2 生成されるパンくず構造

記事タイプに基づいて、以下の構造でパンくずが生成されます：

ノウハウ記事の場合：

ホーム > Howto BOX > ノウハウ記事 > [記事タイトル]

マニュアルの場合：

ホーム > Howto BOX > マニュアル > [記事タイトル]

記事タイトルはdata-title属性から取得されます。属性がない場合はページのh1タグから取得します。

7. 記事テンプレートの使用方法

新しい記事を作成する際は、提供されたテンプレートを使用してください：

  • ノウハウ記事: /howto/article/article-template.html
  • マニュアル: /howto/manual/manual-template.html

7.1 テンプレート使用の手順

  1. 適切なテンプレートファイルをコピーします
  2. 新しい記事IDに基づいて適切なディレクトリを作成します（例：/howto/article/article4/）
  3. コピーしたテンプレートをindex.htmlとしてディレクトリに保存します
  4. 以下の要素を更新します：
    • タイトル、メタ情報
    • 記事ID（data-page属性を設定）
    • 画像パス（記事IDに合わせて修正）
    • 記事コンテンツ

8. 既存コンテンツの移行

既存の記事を新しい構造に移行する際のステップ：

  1. 記事IDを決定します（例：article1, manual2など）
  2. 適切なディレクトリ構造を作成します
  3. 関連する画像ファイルを新しい命名規則に従ってリネームし、適切なディレクトリに移動します
  4. HTML内の画像参照パスを新しいパスに更新します
  5. パンくずリストのマークアップを更新します（data-page属性を設定）

注意: 移行作業は、一貫性を保つために計画的に行ってください。一部の記事だけを移行すると、サイト内で異なる構造が混在することになります。

9. ベストプラクティス

  • 画像は適切な形式を選択してください：
    • 写真やグラデーションが多い画像：JPG形式
    • シンプルなイラストやアイコン、透明度が必要な画像：PNG形式
  • 画像サイズは用途に適したものを選択してください：
    • メイン画像：幅1200px程度
    • セクション画像：幅800px程度
    • サムネイル：幅400px程度
  • すべての画像に適切な代替テキスト（alt属性）を設定してください
  • ファイル名には日本語や特殊文字を使用しないでください

10. まとめ

このガイドラインに従うことで、Howto BOXのコンテンツ管理が効率化され、 統一感のあるユーザー体験を提供できます。適切なディレクトリ構造と命名規則を守ることで、 将来のコンテンツ更新や拡張も容易になります。

不明点がある場合は、Web開発チームにお問い合わせください。
