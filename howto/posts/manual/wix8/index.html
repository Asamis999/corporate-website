このページでできること

  • ブログ/ニュース/お知らせを1つの手順で作成・公開できる
  • 支給されたHTMLデザインフレームを本文に貼り付け、文字と画像だけ差し替えられる
  • 記事タイプ/カテゴリ/タグを設定して一覧で自動出し分けできる

使う場所：Wix ダッシュボード → ブログ（Blog） ／ 記事エディタ

1. 作成フローの全体像

  1. 新しい記事を作成（タイトル→本文）
  2. 本文にHTMLデザインフレームを貼り付け → 文字・画像だけ差し替え
  3. 記事タイプ/カテゴリ/タグを設定（ニュース/お知らせ含む）
  4. 公開または予約投稿 → 一覧ページに反映を確認

✅ ポイント：デザインはフレームを壊さないのがコツ。装飾は最小限。

2. 新しい記事を作成

手順（全3ステップ）

❶ ダッシュボードの ［ブログ］→［記事］→ ［新しい記事］をクリック
❷ タイトル を入力（検索やSNSで読まれる見出し）
❸ 本文 の先頭に、導入文（1〜2行）を書いておく（あとでHTMLフレームを挿入）

3. HTMLデザインフレームを利用した編集

支給されたHTMLコードを本文に貼ります。崩れ防止のためクラス名/入れ子は変更しません。

手順（全3ステップ）

❶ デザイン済みの記事を複製
❷ タイトル/本文のテキスト、画像を差し替え（画像のアップロードは基本操作のマニュアルを確認）
❸ 編集したいアイテムを選択し、ツールバーから[HTMLを編集]をクリック
❹ HTMLの中身を変更（リッチテキスト内で変更したいテキストだけを編集）

⚠ 注意：Wixの環境によっては記事本文にHTML直接貼付が制限される場合があります。その際はHTMLコードブロックの有効化やテンプレート側のウィジェット対応が必要です（専門対応）。

💡 崩れ防止ルール

  • HTML内のクラス名/入れ子は消さない・変えない
  • 画像は同じ比率/近いサイズで差し替え
  • ボタンのリンク先だけ更新（スタイルは触らない）

4. 記事タイプ/カテゴリ/タグの設定（ニュース/お知らせ含む）

記事の種類と分類を設定します。一覧ウィジェットはカテゴリで自動出し分けされます。

手順（全3ステップ）

❶ 記事エディタの ［設定］→［カテゴリ］ を開く
❷ カテゴリー名を選択または新規追加（1〜2個設定するのが推奨）
❸ 次にタグを設定（タグは検索補助）

💡 使い分けの目安

    {{ ... }}
  • お知らせ：公開日が重要（営業時間変更、休業案内、重要告知）
  • ニュース：リリース・イベント告知・メディア掲載
  • ブログ：検索流入向けの解説/読み物（中長文）

5. アイキャッチ/抜粋/URL/公開設定（任意）

サイトの設計次第で使わない項目もあります。利用可能なら以下を設定。

  • カバー画像（アイキャッチ）：一覧カード用。ロゴ+短い見出しで可視性UP
  • 公開/予約投稿：日時を指定して自動公開
  • 抜粋（サマリー）：一覧で表示する短い説明（50〜90字）
  • URLスラッグ：英数字-ハイフン。短く（例：organic-post）
  • メタスクリプションを任意で変更

（検索結果に表示させる内容はサービスやサイトの目的に合わせて設定するとよいでしょう）

6. 反映確認（一覧/詳細）

  1. サイトの一覧ページで、該当カテゴリに記事が出ているか確認
  2. カテゴリフィルタで正しく絞り込めるかテスト
  3. 詳細ページのレイアウト崩れ・リンク切れをチェック

7. チェック

  • フレームのクラス名/入れ子は変えていない？
  • 画像は比率を合わせ、Altテキストを入れた？
  • 記事タイプとカテゴリは正しく選んだ？
  • 公開/予約の設定は合っている？
  • 一覧で出し分けが想定通りになっている？

8. よくあるミスと対処

  • HTMLを整形して崩れた → 貼り付け直し、テキストと画像だけ差し替える
  • 画像が大きすぎて重い → 1200px程度/JPG推奨、2MB以下に
  • カテゴリ未設定で一覧に出ない → 記事タイプ/カテゴリを必ず選択
  • 予約投稿が公開されない → タイムゾーンと日時を再確認
